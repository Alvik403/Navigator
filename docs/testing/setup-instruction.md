# MAX RASS — инструкция первичной настройки

Пошаговый регламент: **2 СМУ + 2 трека + люди + связи + первое занятие**.

Проверено по коду: `app/max-auth/routers/hr.py`, `app/max-auth/domain.py`, `app/web/public/hr-dashboard.html`.

Demo-данные (`POST /api/v1/db/seed-demo` или кнопка на `/db-test`) добавят 3 трека и ~100 пользователей — для «чистой» настройки **не нажимайте** seed.

## 2. Создать 2 СМУ (HR-панель)

**Путь:** `Обучение → СМУ и смены → Новая СМУ`

### СМУ A — пример «2/2»

| Поле | Значение |
|------|----------|
| Код | `smu-a-22` |
| Название | `СМУ A (2/2)` |
| Раб. дней | `2` |
| Выходных | `2` |
| Якорь цикла | дата начала отсчёта, напр. `2026-06-01` |

→ **Создать**

### СМУ B — пример «3/3»

| Поле | Значение |
|------|----------|
| Код | `smu-b-33` |
| Название | `СМУ B (3/3)` |
| Раб. дней | `3` |
| Выходных | `3` |
| Якорь цикла | `2026-06-02` (сдвиг на 1 день — смены не совпадают) |

→ **Создать**

**Как работает якорь:** от `anchor_date` цикл `(work_days + off_days)` повторяется. На формирование попадают только те, у кого **рабочий день** на дату занятия (`domain.is_user_working_on_date`).

**Назначение сотрудникам:** выберите СМУ слева → справа «Доступные» → **+** у нужного ученика. У одного сотрудника — **одна** СМУ (`user_smu` — PK по `user_id`).

**Допсмена:** блок «Дополнительные смены» внизу вкладки — дата + ученик, если человек вышел вне графика.

### API (альтернатива)

```http
POST /api/v1/hr/smu-patterns
Authorization: Bearer <token>
Content-Type: application/json

{
  "code": "smu-a-22",
  "name": "СМУ A (2/2)",
  "work_days": 2,
  "off_days": 2,
  "anchor_date": "2026-06-01"
}
```

## 4. Люди (минимальный состав)

Порядок: **куратор → инструктор → ученики**.

### 4.1 Куратор

**Путь:** `Люди → Кураторы → Добавить`

| Поле | Обязательно | Пример |
|------|-------------|--------|
| Фамилия, имя | да | Петров Павел |
| ID MAX | **да** (уведомления) | реальный числовой ID из бота |
| Телефон | нет | +79001234567 |

Импорт: **Скачать шаблон** → заполнить → **Импортировать**.

### 4.2 Инструктор

**Путь:** `Люди → Инструкторы → Добавить`

| Поле | Обязательно |
|------|-------------|
| ФИО | да |
| ID MAX | желательно |
| Keycloak | да, для веб-панели преподавателя |

**Keycloak** (http://localhost:5173/admin/ → realm `max-education`):

1. Users → Create user → Username, напр. `teacher.ivanov`
2. Credentials → Set password (можно Temporary)
3. Role mapping → роль `teacher`

### 4.3 Ученики (≥4 для теста формирования)

**Путь:** `Люди → Ученики → Добавить`

| Поле | Обязательно для автоформирования |
|------|----------------------------------|
| Фамилия, имя | да |
| ID MAX | да (напоминания в MAX) |
| Куратор | да (ФИО или UUID) |
| Статус | `активен` / `active` |

Шаблон Excel: колонки **Фамилия**, **Имя**, **Отчество**, **ID MAX**, **Телефон**, **Статус**, **Куратор**.

## 5. Связи (без этого формирование не сработает)

### 5.1 Инструктор ↔ треки

**Путь:** `Люди → Инструкторы` → выбрать инструктора → справа список треков → добавить **Бетонщик** и/или **Арматурщик** → **Сохранить**.

API: `POST /api/v1/hr/instructors/{teacher_id}/tracks/sync` с `{ "track_ids": ["uuid1", "uuid2"] }`.

Автоформирование берёт **первого** активного инструктора на треке (`resolve_track_instructor`).

### 5.2 Ученик ↔ трек

**Путь:** `Обучение → Треки` → выбрать трек → **Назначить сотрудника** → выбрать ученика.

Часть учеников — на трек 1, часть — на трек 2; 1–2 человека — на **оба** трека (параллельное обучение поддерживается).

API: `POST /api/v1/hr/users/{user_id}/tracks` → `{ "track_id": "<uuid>", "status": "active" }`.

### 5.3 Ученик ↔ СМУ

**Путь:** `Обучение → СМУ и смены` → выбрать `smu-a-22` → назначить половину учеников; `smu-b-33` → вторую половину.

### 5.4 Куратор ↔ подопечные (опционально, для MAX)

**Путь:** `Люди → Кураторы` → карточка куратора → синхронизация подопечных.

При создании ученика поле **Куратор** уже заполняет `profiles.id_curator`.

## 6. Проверка формирования

**Путь:** `Обучение → Формирование`

1. **Трек** — Бетонщик  
2. **Дата** — завтра (или день, когда у выбранных учеников **рабочая** смена по СМУ)  
3. **Слот** — 09:00  
4. **Тип** — Практика  
5. **Макс. участников** — 12  

→ **Предпросмотр**

Ожидаемо: список **selected** и **excluded** с причинами:

| Причина | Что сделать |
|---------|-------------|
| выходной по СМУ | другая дата или допсмена |
| программа завершена | увеличить `practice_required` или снять лишние отметки |
| просрочен срок программы | продлить `due_date` или переназначить трек |
| недавно назначен (lock) | подождать 14 дней или другой трек |

→ **Сформировать занятие** — создастся запись в `lessons` + участники.

**Авто на все треки/слоты:** кнопка **Автоформирование** на выбранную дату (аналог `POST /api/v1/hr/formation/auto-run`).

Фоновый worker (`FORMATION_AUTO_ENABLED=true`) каждый день **после 18:00 МСК** формирует занятия на **завтра** (`FORMATION_DAYS_AHEAD=1`).

## 7. Расписание и напоминания

### Расписание

**Путь:** `Обучение → Расписание` — проверить созданные занятия (трек, слот, инструктор, участники).

### Напоминания

Worker шлёт в MAX:

| Тип | Когда |
|-----|-------|
| `lesson_reminder_1d` | за ~24 часа |
| `lesson_reminder_3h` | за ~3 часа |

Условия: у ученика `max_id` в профиле; если нет — уведомление уходит **куратору** с `max_id`.

Проверка вручную:

```http
POST http://localhost:8025/api/v1/db/lesson-reminders/run
```

Для теста создайте занятие с `starts_at` = now + 24h (±2 мин окно).

**Путь:** `Контроль → Уведомления` — журнал отправок.

## 8. После занятия

**Путь:** `Обучение → Посещаемость` → отметить `present` / `late` / `absent`.

Без отметок прогресс по треку (`practice_done`) не растёт — человек снова попадёт в формирование.

**Путь:** `Контроль → Страйки` — при нарушениях.

## 9. Чеклист готовности

- [ ] Стенд запущен, вход HR работает  
- [ ] 2 СМУ созданы (или используются `smu-1` / `smu-2` из миграции)  
- [ ] 2 трека созданы с нормами практик/лекций  
- [ ] ≥1 куратор с MAX ID  
- [ ] ≥1 инструктор с привязкой к каждому треку + Keycloak `teacher`  
- [ ] ≥2 ученика на каждый трек, статус active  
- [ ] Ученикам назначены СМУ и кураторы, заполнен MAX ID  
- [ ] Предпросмотр формирования не пустой на рабочий день  
- [ ] Занятие создано, видно в Расписании  
- [ ] (опц.) Напоминания: `LESSON_REMINDER_ENABLED=true` в `.env`

## 10. Переменные окружения (`.env`)

```env

FORMATION_AUTO_ENABLED=true
FORMATION_DAYS_AHEAD=1
FORMATION_AUTO_HOUR_LOCAL=18
LESSON_REMINDER_ENABLED=true
```

Legacy-вход HR/teacher через MAX без записи в БД (только dev):

```env
MAX_USER_MAP_JSON={"1001":{"username":"hr.manager","password":"hr123456","label":"HR","role":"hr_manager"}}
```

Для реальных пользователей — **MAX ID в профиле БД**, не в `MAX_USER_MAP_JSON`.

## 11. Быстрый сценарий «с нуля за 30 минут»

1. Запустить Docker, войти как `hr.manager`.  
2. **СМУ:** создать `smu-a-22` и `smu-b-33` (или использовать встроенные `smu-1` / `smu-2`).  
3. **Треки:** `betonshchik` (12 пр.) и `armaturschik` (8 пр., 2 лек.).  
4. **Куратор** + **инструктор** (Keycloak) + **4 ученика** (импорт Excel).  
5. Инструктору — оба трека; ученикам — треки и СМУ; у всех MAX ID.  
6. **Формирование** → предпросмотр на завтра → сформировать.  
7. **Расписание** — проверить.  
8. После занятия — **Посещаемость**.
