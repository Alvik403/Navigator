MAX RASS — единая схема базы данных

Схема фиксирует пользователей, профили, группы, занятия, посещаемость, страйки и уведомления. Keycloak остаётся внешней системой авторизации, PostgreSQL хранит бизнес-данные приложения.

3. Бизнес-логика

ПрофильОдин users имеет один profiles. Куратор хранится в profiles.id_curator.
Занятиеlessons содержит группу, преподавателя, дату/время, место и тип lecture/practice.
Посещаемостьlesson_members задаёт состав, attendance_marks хранит одну отметку на участника занятия.
СтрайкиАвто-страйк создаётся по absent/late; ручной страйк создаётся с текстовой причиной без обязательного занятия.
УведомленияВ БД хранится только delivered_to: пользователь с телефоном или куратор пользователя без телефона.

4. DDL и реализация

Рабочий SQL лежит в app/max-auth/sql/001_app_schema.sql. FastAPI применяет его на старте сервиса, поэтому схема создаётся автоматически после запуска docker compose.

cd app
docker compose up -d --build

# Проверка API
curl http://localhost:5173/api/v1/db/health
curl http://localhost:5173/api/v1/db/tables
Тестовая страница доступна по адресу http://localhost:5173/db-test. Она разрешает только read-only SELECT/WITH запросы.
