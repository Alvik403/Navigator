1. Общий подход

KeycloakЛогин, пароль, роли доступа. В БД паролей нет.
PostgreSQLПрофили, группы, занятия, страйки, уведомления.
MAX-ботРегистрация, уведомления, страйки. max_user_id в профиле.
Одна БДНа старте одна schema app, без микросервисных схем.
Принцип разделения: таблица users — только связь с Keycloak. Таблица profiles — все бизнес-данные: ФИО, роль, MAX, телефон, куратор.

4. Пояснения и бизнес-логика

Статусы профиля (profiles.status)

СтатусЗначениеЧто можно
pendingОжидает подтверждения HRНичего — HR должен проверить и активировать
activeАктивен, работаетГруппы, занятия, уведомления, посещаемость
inactiveУшёл из программыТолько просмотр истории, без новых действий
bannedЗаблокирован (3 страйка)Ничего — только HR может разбанить

Регистрация через MAX

  1. Пользователь нажимает «Регистрация» в MAX-боте.
  2. Система создаёт users + profiles со статусом pending.
  3. Заполняется max_user_id из webhook MAX.
  4. HR проверяет данные → меняет статус на active, назначает направление и куратора (если нужно).

Сотрудник без MAX и телефона

HR создаёт профиль вручную: max_user_id = NULL, phone = NULL, назначает curator_id. Все уведомления идут куратору с пометкой via_curator = true.

Посещаемость

Преподаватель закрывает занятие → для каждого участника создаётся запись в attendance. Если статус absent — автоматически выдаётся страйк (см. раздел 5).

5. Система страйков

1 страйкУведомление: «У вас 1 из 3 страйков»
2 страйкаУведомление: «У вас 2 из 3 страйков»
3 страйкаБан + уведомление: «Доступ заблокирован»

Когда выдаётся страйк

СобытиеСтрайк?
absent — не пришёл✓ Да
late — опоздал✗ Нет (на старте)
excused — уважительная причина✗ Нет
present — пришёл✗ Нет

Алгоритм

Преподаватель закрывает занятие
  → для каждого absent (не excused):
      1. Посчитать активные страйки (is_revoked = false)
      2. Если уже banned — пропустить
      3. INSERT strikes, strike_number = count + 1
      4. Отправить уведомление «У вас N из 3»
      5. Если strike_number = 3:
           profiles.status = banned
           profiles.banned_at = now()
           Отправить уведомление о бане

Тексты уведомлений

ШаблонТекст
strike_warning Вам выдан страйк за непосещение занятия «{тема}» ({дата}). У вас {N} из 3 страйков. При 3 страйках доступ будет заблокирован.
strike_ban У вас 3 из 3 страйков. Доступ к занятиям заблокирован. Обратитесь к HR или куратору.
Разбан: только HR вручную. HR может отменить страйк (is_revoked = true) — если была уважительная причина.

6. Уведомления и куратор

Куда отправлять

1. profile.max_user_id есть?  → MAX напрямую
2. profile.phone есть?        → SMS
3. profile.curator_id есть?   → MAX/SMS куратору (via_curator = true)
4. Иначе                      → status = failed, HR видит проблему

Пример: сотрудник без MAX, с куратором

7. Ролевая модель

Роли хранятся в Keycloak (доступ к сайтам) и дублируются в profiles.role (бизнес-логика).

admin Полный доступ ко всему
hr Управление сотрудниками и группами
teacher Свои группы и занятия
curator Получает уведомления за подопечных
employee Сотрудник / участник группы
